# 🏫 ระบบติดตามแผนงานโครงการ โรงเรียนครุศาสตร์ (Executive Dashboard)

เว็บแดชบอร์ดติดตามความคืบหน้าของโครงการและแผนงบประมาณสำหรับผู้บริหาร โรงเรียนครุศาสตร์ (คณะครุศาสตร์) ดึงข้อมูลแผนงานโครงการจาก Google Sheets โดยตรงแบบเรียลไทม์เมื่อผู้บริหารเข้าใช้งานหน้าเว็บ

---

## 🎨 โทนสีและแนวคิดการออกแบบ (Red, Blue, Gold Theme)
การออกแบบเน้นความหรูหรา น่าเชื่อถือ และพรีเมียม (Premium Executive Design):
*   **น้ำเงินเข้ม (Navy/Royal Blue)**: เป็นสีพื้นหลังและสีหลัก แสดงถึงความน่าเชื่อถือและความมั่นคง
*   **แดงทับทิม (Ruby Red)**: ใช้สำหรับเน้นส่วนสำคัญ ปุ่มดำเนินการ และแสดงสถานะงบประมาณที่ใช้ไป
*   **ทองเมทัลลิก (Metallic Gold)**: ใช้ตกแต่งขอบ โลโก้ และสถานะหรือข้อความเด่น แสดงถึงความสำเร็จและความพรีเมียม

---

## ✨ คุณสมบัติของระบบ (Key Features)
1.  **การดึงข้อมูลแบบเรียลไทม์ (Real-time Google Sheet Sync)**: ทุกครั้งที่ผู้บริหารเปิดหน้าเว็บขึ้นมาหรือกดปุ่มรีเฟรช ระบบจะไปดึงไฟล์ CSV ล่าสุดจาก Google Sheets ทันที
2.  **ตัวชี้วัดประสิทธิภาพหลัก (KPI Summary Cards)**:
    *   จำนวนโครงการทั้งหมด
    *   งบประมาณจัดสรรทั้งหมด
    *   งบประมาณใช้จริง (พร้อมแสดงสัดส่วนร้อยละของการใช้เงิน)
    *   งบประมาณคงเหลือ (พร้อมแสดงสัดส่วนร้อยละ)
    *   ความคืบหน้าของงานเฉลี่ย
3.  **การแสดงผลเชิงภาพ (Visual Charts)**:
    *   **Doughnut Chart**: สัดส่วนความคืบหน้าแบ่งตามสถานะโครงการ (ยังไม่ดำเนินการ, อยู่ระหว่างดำเนินการ, ดำเนินการแล้ว)
    *   **Bar Chart**: การเปรียบเทียบระหว่างงบประมาณจัดสรรและงบประมาณที่ใช้ไป จำแนกตามกลุ่มงาน
4.  **ระบบกรองข้อมูลอัจฉริยะ (Interactive Search & Filters)**:
    *   ค้นหาโครงการด้วยชื่อโครงการหรือชื่อผู้รับผิดชอบได้ทันที
    *   เลือกกรองตามกลุ่มงาน (กลุ่มงานวิชาการ, สังคมศึกษา, วิทยาศาสตร์ฯ, งบประมาณฯ)
    *   เลือกกรองตามสถานะของโครงการ
5.  **ตารางสรุปผลข้อมูลโครงการ (Detailed Projects Table)**:
    *   แสดงสถานะโครงการด้วยป้ายกำกับสีที่ชัดเจน (Badges)
    *   แสดงแถบความคืบหน้าของแต่ละโครงการ (Progress Bar)
    *   สามารถกดจัดเรียงข้อมูล (Sorting) ตามหัวตารางได้ เช่น เรียงตามงบประมาณ, ความคืบหน้า หรือชื่อผู้รับผิดชอบ
6.  **หน้าต่างแสดงรายละเอียดรายโครงการ (Project Detail Modal)**: คลิกที่แถวโครงการใดๆ เพื่อดูสรุปข้อมูลรายจ่ายและความคืบหน้าอย่างละเอียดแบบการ์ดประมวลผล
7.  **ระบบอัปโหลด-ดาวน์โหลดเอกสารจัดซื้อจัดจ้าง (Procurement Files Manager)**:
    *   **แท็บคลังเอกสาร**: หน้าจอเฉพาะในการแสดงรายการ จัดระเบียบ ค้นหา และอัปโหลดไฟล์เอกสารจัดซื้อจัดจ้างแยกตามโครงการ
    *   **ระบบ IndexedDB**: การใช้ระบบจัดเก็บไฟล์จริงแบบ Binary/Blob ลงในฐานข้อมูล Local บนเบราว์เซอร์อย่างถาวร (ทำให้ใช้งานอัปโหลดและดาวน์โหลดไฟล์บน GitHub Pages ได้โดยตรง)
    *   **การเชื่อมโยงประเภทเอกสาร**: จัดประเภท TOR, PR, PO/สัญญา, ใบเบิกจ่าย/ตรวจรับงาน และมีไอคอนไฟล์สวยงามตามประเภทไฟล์
    *   **แนบไฟล์แบบเร่งด่วน**: สามารถแนบเอกสารและเปิดดาวน์โหลดไฟล์ได้จากหน้าต่างรายละเอียด (Modal) ของแต่ละโครงการได้ทันที
    *   **ลิงก์เอกสารออนไลน์**: รองรับการอ่านลิงก์จากคอลัมน์ที่ 10 ของ Google Sheets หากใส่ลิงก์ Google Drive หรือเว็บไซต์ภายนอก ระบบจะดึงมาทำเป็นปุ่มกดเปิดให้โดยอัตโนมัติ

## 🚀 คู่มือการอัปโหลดขึ้น GitHub และเปิดใช้งาน GitHub Pages

โปรเจกต์นี้เขียนด้วย HTML, CSS และ JavaScript แท้ (Static Web) จึงสามารถโฮสต์ผ่านบริการ **GitHub Pages** ได้ฟรี โดยมีขั้นตอนดังนี้:

### ขั้นตอนที่ 1: การเตรียมโค้ดในเครื่องคอมพิวเตอร์ของคุณ
เปิด Terminal หรือ PowerShell ในโฟลเดอร์นี้ (`c:\Users\KPRU  AI\Documents\pramon`) แล้วรันคำสั่งดังนี้:

1.  **เริ่มต้นใช้งาน Git Repository**:
    ```bash
    git init
    ```
2.  **เพิ่มไฟล์ทั้งหมดเตรียมการ Commit**:
    ```bash
    git add .
    ```
3.  **บันทึกการทำงานครั้งแรก**:
    ```bash
    git commit -m "Initial commit of executive dashboard website"
    ```

---

### ขั้นตอนที่ 2: การสร้าง Repository และอัปโหลดขึ้น GitHub
1.  เปิดเว็บไซต์ [GitHub](https://github.com/) และเข้าสู่ระบบบัญชีของคุณ
2.  คลิกปุ่ม **New** (สร้าง Repository ใหม่)
3.  ตั้งชื่อ Repository เช่น `pramon-school-dashboard`
4.  เลือกสถานะเป็น **Public** (จำเป็นต้องเลือกเพื่อให้ใช้ GitHub Pages แบบฟรีได้)
5.  **ไม่ต้องติ๊ก** เลือก Add a README file, Add .gitignore หรือ Choose a license (ปล่อยว่างไว้) แล้วคลิกปุ่ม **Create repository**
6.  คัดลอกคำสั่งในส่วน "…or push an existing repository from the command line" มาวางและรันในหน้าต่าง Terminal/PowerShell ของคุณ (เปลี่ยน URL ตามบัญชีของคุณ):
    ```bash
    # เปลี่ยนชื่อกิ่งหลักเป็น main
    git branch -M main
    
    # เชื่อมโยงโฟลเดอร์ในเครื่องกับ GitHub (แทนที่ลิงก์ด้านล่างด้วยลิงก์ repository ของคุณ)
    git remote add origin https://github.com/ชื่อผู้ใช้GitHubของคุณ/pramon-school-dashboard.git
    
    # อัปโหลดไฟล์ขึ้น GitHub
    git push -u origin main
    ```

---

### ขั้นตอนที่ 3: เปิดใช้งาน GitHub Pages เพื่อเผยแพร่เว็บไซต์
เมื่ออัปโหลดไฟล์เสร็จสิ้นแล้ว ให้ทำตามขั้นตอนนี้ในเบราว์เซอร์บนหน้า GitHub Repository:

1.  ที่หน้า Repository ของคุณ ให้คลิกเมนู ⚙️ **Settings** (การตั้งค่า) ที่แถบเมนูด้านบน
2.  ที่แถบเมนูด้านซ้าย ในส่วนของ **Code and automation** ให้เลือกเมนู 📄 **Pages**
3.  ในหัวข้อ **Build and deployment**:
    *   **Source**: เลือกเป็น `Deploy from a branch` (ค่าเริ่มต้น)
    *   **Branch**: เปลี่ยนจาก `None` เป็น `main` และเลือกโฟลเดอร์เป็น `/ (root)`
4.  คลิกปุ่ม **Save**
5.  รอประมาณ 1 - 2 นาที ให้ทำการรีเฟรชหน้าเว็บนั้น ด้านบนสุดจะมีแถบสีเขียวแสดงข้อความลิงก์ของเว็บไซต์ เช่น:
    > **Your site is live at** `https://ชื่อผู้ใช้GitHubของคุณ.github.io/pramon-school-dashboard/`
6.  คุณสามารถคัดลอกลิงก์ดังกล่าวส่งต่อให้ผู้บริหารหรือนำไปฝังในเว็บไซต์หลักของโรงเรียนได้ทันที!

## 📊 ข้อมูลสำหรับการเชื่อมต่อ Google Sheets
หากในอนาคตต้องการเปลี่ยนชุดข้อมูล หรือสร้างตารางใหม่ ให้คัดลอกข้อมูลลง Google Sheet ใหม่ แล้วแก้ไขตัวแปร `SHEET_CSV_URL` ในไฟล์ `app.js` เป็นลิงก์ใหม่ในรูปแบบเดียวกัน:
```javascript
const SHEET_CSV_URL = "https://docs.google.com/spreadsheets/d/รหัสตารางอันใหม่ของคุณ/export?format=csv";
```
*หมายเหตุ: ต้องตั้งค่าสิทธิ์ให้ Google Sheet นั้นเป็น "ทุกคนที่มีลิงก์สามารถอ่านได้" (Anyone with the link can view) เพื่อให้ระบบสามารถเข้าถึงได้*
